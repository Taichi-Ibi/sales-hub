# Sales Hub（営業ナレッジベース）— モックアップ

社内営業組織向けプロダクトの**クリッカブルUIモックアップ**です。
見た目と画面遷移のみを再現します。バックエンド・AI・メール連携は実装していません
（すべてモックデータと擬似挙動）。

## コンセプト — raw → wiki → ビュー

Karpathy の「LLM Knowledge Base」パターンを組織向けに拡張した3層構造です。

| 層 | 画面 | 役割 |
| --- | --- | --- |
| raw | 受信箱 `/inbox` | Slack/メール/議事録の原文。ローカル前処理（自動マスク・案件判定）まで自動。**AIへの送信は全件、人の承認が必要（HITL）** |
| wiki | 案件 `/projects` | 1案件=1ページをAIが維持。全記述に**出典**、毎朝の**整合性チェック（lint）**、**質問応答** |
| ビュー | 今日 `/`（ホーム） | AIが抽出した「次の一手」。やる/待ち/済みの3タブ。開く→下書きコピー→送信の3手で完了 |

設計原則は「**AIが処理し、境界は人が握る（HITL）**」。前処理が下ごしらえを済ませ、
人は AI送信前レビューで確認・承認します。承認の記録（誰が・いつ）は監査ログとして残ります。

## 技術スタック

- Vite + React + TypeScript + Tailwind CSS v4
- ルーティング: React Router（`HashRouter` — GitHub Pages のサブパス直リンク対策）
- 状態管理: React の `useState` / Context のみ（外部ライブラリ不使用）
- ホスティング: GitHub Pages（`main` への push で自動ビルド・デプロイ）

## セットアップ

```bash
npm install      # 依存をインストール
npm run dev      # 開発サーバ（http://localhost:5173）
npm run build    # 本番ビルド（dist/ に出力）
npm run preview  # ビルド結果をローカル確認
npm run typecheck
```

## デモ操作（モック専用）

- **受信箱 → AI送信前レビュー（警告あり）**: 「解約条項の見直しのお願い」を開く →
  本文の「川島」をタップして氏名マスク → 「承認してAIに渡す ▶」→ タスク生成＋案件ページに取込履歴が追記される
- **受信箱 → AI送信前レビュー（案件不明）**: 「#sales-相談」を開く → プロジェクト（湊精機）を選択 → 「承認してAIに渡す ▶」
- **受信箱 → AI送信前レビュー（警告なし）**: 「#sales-general 進捗共有」を開く → 内容確認 → 承認 → 解析結果は「タスクなし」
- **案件ページ**: 出典チップ 📎 から原文へ遷移／「この案件に質問する」で質問チップをタップ
- **今日 → 低リスク詳細**: 「承認して送信」→ 済みへ
- **今日 → 高リスク詳細**: 「FS承認へ回す」→ 待ちへ →「（デモ）FSが承認する」→「送信する」

## ディレクトリ

```
src/
  data/inbox.ts          # raw層: 受信箱（原文・自動マスク・承認前警告）
  data/wiki.ts           # wiki層: 案件ページ（出典つき記述・更新履歴・lint・想定問答）
  data/actions.ts        # ビュー: タスク
  data/deals.ts          # 登録済みプロジェクト台帳（設定・ピッカー用）
  types.ts               # ドメイン型
  lib/time.ts            # 経過時間の算出・色分け
  store/StoreContext.tsx # 全状態と状態遷移（承認→AI解析→タスク化→wiki追記）
  components/             # Shell / カード / バッジ / 下書き / ドリルダウン 等
  pages/                  # 今日 / 案件 / 受信箱 / 設定 の各画面
```

> 会社名・人名はすべて架空です。実在の組織・個人は使用していません。
