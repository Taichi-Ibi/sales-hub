# Sales Hub — 医療システムの良さを取り入れた営業CRM 設計メモ

## コンセプト

電子カルテ（EHR）／病院情報システムが長年磨いてきた「人の状態を見落とさず、
チームで継続的にケアする」仕組みを、営業の商談管理に移植する。

商談を「患者」、営業を「主治医」、CRM を「カルテ」と見立てる。

## 医療の良い点 → 営業CRMへの翻訳

| 医療システムの良い点                       | 営業CRMへの応用                                         | 本モックでの実装               |
| ------------------------------------------ | ------------------------------------------------------- | ------------------------------ |
| トリアージ（緊急度で色分け・優先順位付け） | 商談トリアージ：赤=要対応 / 黄=注意 / 緑=順調           | ダッシュボード・一覧の色分け   |
| バイタルサイン監視                         | 商談バイタル：熱量・モメンタム・鮮度・決裁者接触        | カルテ上部のバイタルモニター   |
| 異常値アラート                             | 失注リスクアラート（接触途絶・停滞・決裁者未接触）      | ダッシュボードのアラート欄     |
| SOAP記録（構造化された診療記録）           | 商談メモを S/O/A/P で構造化して記録                     | カルテのタイムライン記録       |
| 既往歴・アレルギー警告（常時表示）         | アカウント注意フラグ（価格敏感・競合利用・過去解約 等） | カルテ最上部の常時バナー       |
| タイムライン型カルテ                       | 全接点（訪問・電話・メール）を時系列で一元化            | カルテのタイムライン           |
| クリニカルパス（標準診療計画）             | セールスパス（標準商談ステージ進行）                    | カルテのステージプログレス     |
| 申し送り（シフト引き継ぎ）                 | 担当引き継ぎ・チーム共有ノート                          | カルテのサマリ                 |
| リコール（定期再診の呼び戻し）             | フォローアップ・次回アクション期日                      | バイタル「鮮度」＋次アクション |

## 画面構成

1. **ダッシュボード（ナースステーション）** `/`
   - 本日のトリアージ：要対応の商談を緊急度順に
   - 全体バイタル（KPI）
   - 失注リスクアラート

2. **商談一覧（病棟一覧）** `/deals`
   - トリアージ色・バイタル要約付きの一覧
   - ステージ・担当でのフィルタ

3. **商談カルテ（電子カルテ）** `/deals/[id]` ← 中核
   - 注意フラグバナー（アレルギー警告相当）
   - バイタルモニター
   - セールスパス（クリニカルパス相当）
   - SOAP タイムライン

## 設計原則

- **見落とさない**: 異常（停滞・接触途絶）は色とアラートで能動的に通知。
- **一元化**: 1商談=1カルテ。全接点が時系列で残る。
- **構造化**: メモは SOAP で「事実／解釈／次の一手」を分離。
- **継続性**: 担当が変わっても申し送りで文脈が引き継がれる。

## 技術

- SvelteKit (Svelte 5 runes) + adapter-static、プリレンダリング。
- 追加依存なし。デザインシステムは自前 CSS（`src/lib/styles/`）。
- モックデータは `src/lib/data/`。基準日は 2026-06-02。
