# Sales Hub — 顧客中心 × イベントソーシング CRM 設計メモ

## コンセプト

「案件」ではなく「顧客」を最上位エンティティに据え、イベントソーシングによって
顧客とのあらゆる接点を時系列で管理する。現在の状態（案件ステータス・売上累計など）は
保持せず、積み上げたイベント履歴から都度導出する。

着想は電子カルテ：

| カルテの考え方         | 本CRMでの対応        |
| ---------------------- | -------------------- |
| 患者                   | 顧客（Customer）     |
| カルテ（時系列の記録） | イベントタイムライン |
| 診療・治療             | 案件（Project）      |

ただし UI は医療色を出さず、一般的なモダン SaaS CRM の見た目に統一している。

## アーキテクチャ

```
Customer（顧客・最上位）
 ├ Project（案件A）
 ├ Project（案件B）
 └ SalesEvent[]（全接点のイベント履歴）
```

- **Customer** … 会社情報・担当営業・顧客ランク・注意事項フラグ・担当者。
- **Project** … 顧客配下の案件。想定売上 / 想定粗利率を持つ。ステータスは持たない。
- **SalesEvent** … 唯一の真実。顧客イベント（登録・接触・電話・訪問・定例会 等）と
  案件イベント（案件作成・提案提出・見積提出・受注・失注・開発開始・納品 等）。

### イベントからの状態導出（`src/lib/data/utils.ts`）

- `deriveProjectStatus(projectId)` … 状態遷移イベントの最新値から案件ステータスを導出。
- `customerRevenue(customerId)` … 受注イベントの金額合計（＝LTV相当）。
- `customerMargin(customerId)` … 受注金額で加重平均した粗利率。
- `dashboardMetrics()` … KGI / KPI / 補助指標を今期イベントから集計。

## 画面構成

1. **ダッシュボード** `/`
   - KGI：売上 / 粗利率
   - KPI：商談数 / 提案数 / 受注率 / 平均案件単価
   - 補助指標：新規顧客数 / アクティブ顧客数 / パイプライン金額
   - 全社の最新イベントフィード・要フォロー顧客・売上累計上位

2. **顧客一覧** `/customers`
   - ランク別フィルタ。売上累計・粗利率・進行中案件・最終接触を一覧。

3. **顧客詳細（中核）** `/customers/[id]`
   - ヘッダー：会社名・ランク・売上累計・粗利率・担当営業
   - 注意事項バナー（常時表示）
   - **イベントタイムライン**（顧客×案件の全接点を時系列／イベント追記フォーム）
   - 案件一覧・担当者・基本情報

4. **案件一覧** `/projects`
   - 進行中 / 受注 / 失注フィルタ。ステータスは導出値。

5. **案件詳細** `/projects/[id]`
   - 概要・売上予測 / 粗利率・ステータス（導出）・案件タイムライン
   - 提案書 / 見積書 / 契約情報（イベントの添付・受注ペイロードから生成）

## 設計原則

- **顧客中心**：案件が終わっても顧客との関係履歴は途切れない。
- **イベントソーシング**：状態を持たず履歴から導出。監査性・分析基盤への発展性が高い。
- **顧客深耕の可視化**：1社の複数案件・リピート・LTV を顧客単位で俯瞰できる。
